# StoryForge 技术方案

## 1. 技术栈选型

### 前端
- **框架**: React 18 + TypeScript
- **构建工具**: Vite 5
- **UI 库**: 待定 (建议 TailwindCSS + shadcn/ui 或 Radix)
- **状态管理**: Zustand (轻量) 或 React Query
- **路由**: React Router v6
- **HTTP 客户端**: Axios 或 fetch wrapper

### 后端
- **运行时**: Node.js 20 LTS
- **框架**: Express.js 或 Fastify
- **语言**: TypeScript
- **ORM**: Prisma (支持 SQLite/PostgreSQL)
- **认证**: JWT (jsonwebtoken) + bcrypt

### 数据库
- **开发环境**: SQLite (零配置，易于开发)
- **生产环境**: PostgreSQL 16 (生产级可靠性)
- **迁移**: Prisma Migrate

### 部署
- **容器化**: Docker + Docker Compose
- **Web 服务器**: Nginx (静态文件 + 反向代理)
- **CI/CD**: GitHub Actions

## 3. 数据库 Schema 设计

### 核心实体

#### User (用户)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| username | String | 用户名 (唯一) |
| email | String | 邮箱 (唯一) |
| password | String | 密码 (bcrypt 加密) |
| createdAt | DateTime | 创建时间 |
| updatedAt | DateTime | 更新时间 |

#### Project (项目)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| name | String | 项目名称 |
| description | String? | 项目描述 |
| userId | UUID | 所属用户 (外键) |
| createdAt | DateTime | 创建时间 |
| updatedAt | DateTime | 更新时间 |

#### Novel (小说)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| title | String | 小说标题 |
| genre | String? | 类型/题材 |
| synopsis | String? | 简介 |
| projectId | UUID | 所属项目 (外键) |
| createdAt | DateTime | 创建时间 |
| updatedAt | DateTime | 更新时间 |

#### Character (角色)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| name | String | 角色名称 |
| description | String? | 角色描述 |
| personality | String? | 性格特点 |
| appearance | String? | 外貌特征 |
| background | String? | 背景故事 |
| novelId | UUID | 所属小说 (外键) |
| createdAt | DateTime | 创建时间 |
| updatedAt | DateTime | 更新时间 |

#### Chapter (章节)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| title | String | 章节标题 |
| content | Text | 章节内容 |
| order | Int | 章节顺序 |
| wordCount | Int | 字数统计 |
| status | Enum | 状态 (draft/published/archived) |
| novelId | UUID | 所属小说 (外键) |
| createdAt | DateTime | 创建时间 |
| updatedAt | DateTime | 更新时间 |

## 4. API 设计

### 认证接口
- `POST /api/auth/register` - 用户注册
- `POST /api/auth/login` - 用户登录
- `GET /api/auth/me` - 获取当前用户

### 项目接口
- `GET /api/projects` - 获取用户项目列表
- `POST /api/projects` - 创建项目
- `GET /api/projects/:id` - 获取项目详情
- `PUT /api/projects/:id` - 更新项目
- `DELETE /api/projects/:id` - 删除项目

### 小说接口
- `GET /api/projects/:projectId/novels` - 获取小说列表
- `POST /api/projects/:projectId/novels` - 创建小说
- `GET /api/novels/:id` - 获取小说详情
- `PUT /api/novels/:id` - 更新小说
- `DELETE /api/novels/:id` - 删除小说

### 角色接口
- `GET /api/novels/:novelId/characters` - 获取角色列表
- `POST /api/novels/:novelId/characters` - 创建角色
- `PUT /api/characters/:id` - 更新角色
- `DELETE /api/characters/:id` - 删除角色

### 章节接口
- `GET /api/novels/:novelId/chapters` - 获取章节列表
- `POST /api/novels/:novelId/chapters` - 创建章节
- `PUT /api/chapters/:id` - 更新章节
- `DELETE /api/chapters/:id` - 删除章节

## 5. 部署方案

### 开发环境
```bash
cd server && npm install && npm run dev
cd client && npm install && npm run dev
```

### Docker 部署 (生产环境)
- **Nginx**: 端口 80/443，反向代理到后端，托管前端静态文件
- **Node.js**: 运行后端服务
- **PostgreSQL**: 数据存储
- ** volumes**: 数据持久化

### 环境变量
```
# Server
DATABASE_URL=postgresql://user:pass@db:5432/storyforge
JWT_SECRET=your-secret-key
PORT=3000

# Client
VITE_API_BASE_URL=/api
```

## 6. 风险点与建议

### 风险
1. **AI API 集成**: 尚未确定使用 GPT/Claude API，需后续评估
2. **数据迁移**: SQLite → PostgreSQL 迁移需测试
3. **文件存储**: 大量小说内容存储策略待定

### 建议
1. 初期使用 SQLite 快速开发，后期平滑迁移 PostgreSQL
2. AI API 集成建议封装独立 Service，便于切换provider
3. 考虑使用对象存储 (OSS/S3) 存储导出文件
